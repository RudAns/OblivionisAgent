# 技能 · 报告式 HTML 风格（Claude 暖色书卷风 · 完整版）

生成**报告 / 演示 / 文档型**（非玩法、非游戏化）HTML 时，严格套用以下设计系统（取自 Anthropic Claude 官网）。气质：温暖、克制、像高级纸张上的书卷，而非冷冰冰的科技界面。

## 一、配色（全暖调，任何灰都带黄褐底色，禁冷蓝灰）

**背景 / 表面**
- 羊皮纸 `#f5f4ed` — 页面主底（暖奶油，像旧纸）
- 象牙 `#faf9f5` — 卡片 / 抬升容器
- 纯白 `#ffffff` — 仅特定按钮 / 最高对比元素
- 暖砂 `#e8e6dc` — 按钮底 / 显著交互面
- 深灰面 `#30302e` — 暗主题容器 / 导航边
- 近黑 `#141413` — 暗主题页底 / 主深色面

**文字**
- 近黑 `#141413` — 主文字
- 炭灰暖 `#4d4c48` — 浅暖面上的按钮字
- 橄榄灰 `#5e5d59` — 次要正文
- 石灰 `#87867f` — 三级文字 / 脚注 / 元信息
- 暖银 `#b0aea5` — 暗底上的正文

**品牌 / 语义**
- 赤陶 `#c96442` — 核心品牌色，**只用于主 CTA、品牌时刻、最高信号强调**
- 珊瑚 `#d97757` — 赤陶的更亮变体，暗底链接 / 次强调
- 错误绯红 `#b53333` — 错误态（稳重不刺眼）
- 聚焦蓝 `#3898ec` — **整套唯一的冷色**，仅用于输入框聚焦环（无障碍）

**边框 / 环**
- 奶油 `#f0eee6` — 浅色标准边（最轻的包裹）
- 暖砂 `#e8e6dc` — 显著边 / 分区分隔
- 深 `#30302e` — 暗面边框
- 环色 `#d1cfc5` — 按钮 hover / focus 的环阴影

> 无传统渐变。层次靠暖色调彼此过渡（奶油 → 砂 → 石灰 → 炭 → 黑）+ 明暗分区交替。

## 二、字体

- **标题**：衬线 `Anthropic Serif`（回退 `Newsreader` / `Georgia`），**统一 weight 500**，绝不加粗 >500。
- **正文 / UI**：无衬线 `Anthropic Sans`（回退 `Inter` / `system-ui`）。
- **代码**：等宽（仅限代码）。

**层级（字号 / 行高）**

| 角色 | 字体 | 字号 | 行高 |
|---|---|---|---|
| Hero / 主标 | 衬线 | 64px | 1.10 |
| 区块标题 | 衬线 | 52px | 1.20 |
| 大副标 | 衬线 | 36px | 1.30 |
| 副标 / 卡片标题 | 衬线 | 32px | 1.10 |
| 小副标 | 衬线 | 25px | 1.20 |
| 功能小标题 | 衬线 | 20.8px | 1.20 |
| 衬线正文（编辑段落） | 衬线 | 17px | 1.60 |
| 大正文 / 引言 | 无衬线 | 20px | 1.60 |
| 标准正文 / 按钮字 | 无衬线 | 16px | 1.25–1.60 |
| 说明 / caption | 无衬线 | 14px | 1.43 |
| 标签 / badge | 无衬线 | 12px | 字距 0.12px |

原则：衬线管「权威」（内容标题）、无衬线管「实用」（UI）；衬线全用 500 形成统一嗓音；正文行高松到 **1.60**（书卷感）；标题行高紧 1.10–1.30 但不局促；12px 以下小字加字距 0.12–0.5px。

## 三、组件

**按钮**（圆角 8–12px）
- 主 CTA：赤陶 `#c96442` 底 + 象牙字 + 环阴影 `0 0 0 1px #c96442`（**唯一带彩色的按钮**）
- 次按钮（暖砂）：`#e8e6dc` 底 + 炭灰 `#4d4c48` 字 + 环 `0 0 0 1px #d1cfc5`
- 白面按钮：白底 + 近黑字，hover 转次背景
- 暗炭按钮：`#30302e` 底 + 象牙字（暗底强调用）

**卡片 / 容器**
- 底：象牙 / 白（浅）｜深灰 `#30302e`（暗）
- 边：`1px solid #f0eee6`（浅）｜`1px solid #30302e`（暗）
- 圆角：标准卡 8px｜特色卡 16px｜Hero / 大媒体 32px
- 阴影：极轻 `rgba(0,0,0,0.05) 0 4px 24px`；交互态用环 `0 0 0 1px`
- 列表项分隔：仅上边框 `1px 0 0`

**输入框**：近黑字、紧凑纵向 padding、圆角 12px、聚焦用 `#3898ec` 环（唯一冷色）。
**导航**：sticky 暖底（可加毛玻璃模糊）、logo 近黑、链接近黑 / 橄榄灰、CTA 赤陶或白面、hover 仅变色不加下划线。
**图片**：截图圆角 16–32px；暗色 UI 截图放在浅底分区更跳。

## 四、深度 / 阴影（靠暖环，不靠重投影）

| 级别 | 处理 | 用途 |
|---|---|---|
| 0 平 | 无阴影无边框 | 背景 / 内联文字 |
| 1 包裹 | `1px solid` 暖边 | 标准卡片 / 分区 |
| 2 环 | `0 0 0 1px` 暖灰 | 交互卡 / 按钮 / hover 态 |
| 3 轻浮 | `rgba(0,0,0,0.05) 0 4px 24px` | 抬升的特色卡 / 截图 |
| 4 内凹 | `inset 0 0 0 1px`（15% 透明） | 按下态 |

签名手法：环阴影 `0 0 0 1px`——像「假装成边框的阴影」，比真边框更柔。最戏剧的深度来自**明（羊皮纸）/ 暗（近黑）分区交替**，像翻书的章节。

## 五、布局

- 间距基准 8px；常用尺度 4 / 6 / 8 / 12 / 16 / 20 / 24 / 30。
- 卡片内边距 24–32px；主分区上下留白 **80–120px**。
- 容器最大 ~1200px 居中。Hero 居中编辑式；功能区单列或 2–3 列卡网格。
- 圆角梯度：4（内联）/ 6–8（小按钮 / 卡）/ 12（主按钮 / 输入 / 导航）/ 16（特色容器）/ 24（标签）/ 32（Hero / 大媒体）。
- 整页**明 / 暗分区交替**成「章节」节奏。
- 响应式（要做才管）：桌面 992px+ 全多列、Hero 64px；平板 768–991 起 2 列；手机 <640 单列、Hero 降到 36px 左右、导航收起。

## 六、Do / Don't

**Do**：羊皮纸做主底；衬线统一 500；赤陶只给主 CTA；所有灰带暖调；交互用环阴影；正文行高 1.60；明暗分区交替；圆角 12–32px 偏软。
**Don't**：别用冷蓝灰；衬线别加粗 >500；除赤陶外别上饱和色；按钮 / 卡别用 <6px 尖角；别用重投影；别拿纯白当页底（用羊皮纸 / 象牙）；别用几何科技风插画；正文行高别 <1.4；非代码别用等宽；标题别用无衬线。

> 仅对「报告 / 演示 / 文档型」HTML 生效；玩法 / 游戏化 / 交互玩具型 HTML 不受此约束。
