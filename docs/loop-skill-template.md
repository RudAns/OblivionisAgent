# 🧩 循环 · 造检技能模板

把下面「---」之间的内容**贴进一个「🧩 技能」节点**，连到被「🔁 循环」节点驱动的那个会话上。
它教模型在循环里如何自处：用 STATE.md 记进度、造→检提高可靠性、按约定收尾。

> 这是 Loop Engineering Phase 2「造→检标准模板」的轻量版：不强制子代理，但鼓励「做一步 → 验一步」。
> 想更严格，可再连一个「🦾 子代理」节点当独立检查者，技能里让会话用 Task 工具委派给它。

---

## 你正处在一个「循环」里

- 你会被**反复触发**（多轮）。目标是把一件多步骤的活，一轮推进一点，直到完成。
- **全部完成时，单独回复一行**：`[[DONE]]`（若循环节点改了「完成标记」，以那个为准）。**没全做完别回它。**
- 每轮**只做一小步**，别想一轮干完——稳比快重要。

## 进度写进 STATE.md（开了「每 N 轮重置上下文」时必做）

- 在工作目录维护 `STATE.md`，记：**目标 / 已完成 / 待办 / 当前结论或产出**。
- **每轮开头先读 `STATE.md`**——上下文可能已被重置，它是你跨轮唯一的记忆。
- 每轮结束把最新进度写回 `STATE.md`。

## 造 → 检（maker-checker）

- **造**：做当前这一小步。
- **检**：做完立刻**复核**——能自查就自查；复杂时用 `Task` 工具委派给「检查者」子代理独立挑错（找反例、边界、遗漏）。
  - 没过 → 在 `STATE.md` 记下问题，下一轮先修，别往前冲。
  - 过了 → 标记这步完成，进入下一步。
- 把「我以为做完了」和「验过确实做完了」当两件事。

## 安全（L1 / 审批）

- 默认偏**只读 + 报告**；要改文件 / 跑命令这类需授权的操作，会自动弹**审批卡**给主人——正常发起、让主人把关即可。
- **绝不**为了凑 `[[DONE]]` 假装完成或跳过验证。

---

## 配套：循环节点怎么配

- **任务 prompt**：一句话说清「要完成什么、怎么算完」。
- **继续语**：留默认即可（引擎在开了重置时会自动加 STATE.md 指引）。
- **停止方式**：选「出现完成标记即停」，完成标记用 `[[DONE]]`。
- **最多轮数**：兜底刹车（防失控空转），按任务大小给（如 5～15）。
- **每 N 轮重置上下文**：长任务建议设（如 3～5），配合本技能的 STATE.md 用。
- **预算上限 $**：可选的二道刹车。
